# Documentación: _MANUAL-PROGRAMADOR.MD

## Propósito

El Manual del Programador es el documento principal de onboarding para programadores que se incorporan al proyecto. Su objetivo es permitir que un **nuevo programador con conocimientos de las herramientas utilizadas** (Laravel, React, Vite, Playwright, etc.) pueda:

- Entender el **objetivo funcional** del proyecto
- Comprender la **estructura técnica** completa
- Configurar el entorno y comenzar a trabajar
- Seguir las convenciones y reglas del proyecto

**Sin necesidad de explicación humana.**

## Ubicación

- **Archivo:** `_MANUAL-PROGRAMADOR.MD` (raíz del proyecto)
- **Versión actual:** 1.4
- **Última actualización:** 2025-02-15

## Estructura del Documento

1. **Objetivo Conceptual del Proyecto** – Qué es, propósito, flujo E2E, entidades, principios
2. **Stack Tecnológico** – Backend (Laravel), Frontend (React), BD, DevOps
3. **Ruta de Lectura** – Fases ordenadas (1-8) para entender el diseño completo
4. **Estructura del Proyecto** – Árbol de directorios comentado
5. **Convenciones y Reglas** – Backend, Frontend, BD, Testing
6. **Cómo Empezar a Trabajar** – Configuración, lectura, desarrollo, checklist
7. **Referencias Importantes** – Tabla de documentos clave
8. **Preguntas Frecuentes**
9. **Historial de Actualizaciones**

## Relación con Otros Documentos

- **`_PROJECT_CONTEXT.md`** – Contexto breve; el manual amplía con ruta de lectura
- **`README.md`** – Información general; el manual es la guía de incorporación
- **`AGENTS.md`** – Guía para el agente IA; el manual es para humanos
- **`docs/historias-y-tickets.md`** – Documento central de funcionalidades referenciado en el manual
