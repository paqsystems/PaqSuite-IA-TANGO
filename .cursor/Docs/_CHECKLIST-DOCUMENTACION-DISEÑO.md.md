# Documentación: _CHECKLIST-DOCUMENTACION-DISEÑO.md

## Propósito

Checklist estructurado para el proceso de documentación y diseño de un proyecto MVP. Sirve como:
- **Guía de verificación** para comprobar completitud del proceso de documentación
- **Plantilla reutilizable** para proyectos nuevos
- **Referencia** del proceso seguido en este proyecto

## Ubicación

`_CHECKLIST-DOCUMENTACION-DISEÑO.md` (raíz del proyecto)

## Estructura Principal

**Fases 1-15** cubren:
1. Definición y contexto
2. Flujo E2E y planificación
3. Diseño del modelo de datos
4. Arquitectura
5. Historias de usuario
6. Tickets técnicos
7. Especificaciones técnicas
8. Documentación de producto
9. Configuración de herramientas (testing, CI/CD)
10. Documentación de soporte (Manual, READMEs)
11. Reglas y convenciones
12. Registro de uso de IA
13. Validación y revisión
14. Commit y versionado
15. Documentación de código (PHPDoc, JSDoc)

**Sección adicional:** Checklist paso a paso para proyectos nuevos (según Pablo Quarracino), con comandos de setup y test.

## Relación con Otros Documentos

- Complementa `_MANUAL-PROGRAMADOR.MD` (el manual es para incorporación; el checklist es para el proceso de documentación)
- Referenciado por `docs/historias-y-tickets.md` y `AGENTS.md`
- Los archivos referenciados usan prefijo `_` cuando aplica (ej: `_PROJECT_CONTEXT.md`)
