# description: Reglas del proyecto MVP – Entregables obligatorios
---

Este proyecto corresponde a un MVP académico/profesional con consignas explícitas.

# Reglas para el agente IA:

## Priorizar siempre el flujo E2E definido como MVP.
## Evitar funcionalidades fuera de alcance.
## Todo código debe:
   - Soportar el flujo principal
   - Ser testeable
   - Ser comprensible
## No sobre–diseñar arquitectura ni modelo de datos.
## Recordar que deben existir:
   - Tests unitarios
   - Tests de integración
   - Al menos un test E2E
## Toda decisión relevante debe poder documentarse.
